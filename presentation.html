Challenge 9 — The Scouts

5 Parallel Subagents

One per seam, launched in a single message, fully explicit context in each prompt (subagents don't inherit coordinator context).

  • 3/5 full agreement with ADR-001
  • Both divergences share one root cause

The Divergence

Agents scored technical risk correctly but missed operational overhead and strategic sequencing — dimensions not in the prompt. Two extra scoring criteria would close both gaps.

What Made This Work — Claude Code Features

CLAUDE.md

Three-level context hierarchy

Ground rules written once in Challenge 3 held throughout 7 challenges. Claude never touched the monolith without tests, never suggested copying JPA annotations.

PreToolUse Hook

Hard guardrail vs. soft prompt

Exit code 2 cannot be argued away by context. The fence hook catches authoring mistakes before they reach the filesystem. ADR-002 documents the distinction.

Characterization

Behavior-first discipline

22 tests written against undocumented API in one pass. Claude read the source, found the edge cases (including the 200+empty bug), generated assertions that pinned all of them.

Subagents

Parallel fan-out analysis

5 independent risk assessments in one message, ~30 seconds. The divergences from the human ADR were as informative as the agreements — revealed exactly which dimensions require explicit prompting.

Eval Harness

Scorecard with real metrics

Claude CLI fallback means the eval runs in any Claude Code session without managing API keys. Accuracy and false-confidence rate are defensible numbers, not vibes.

Key Insight

The monolith didn't change.
The safety net did.

Characterization tests, fence hooks, contract tests, and a rehearsed runbook don't speed up extraction — they make it safe to do at all. Claude Code enforces the rules that a tired developer at 3am would otherwise skip.

Hooks enforce. Prompts prefer. Pin bugs before you fix them. Subagents diverge informatively.
